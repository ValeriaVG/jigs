# cf-rag — agentic RAG pipeline (example)

> This example (code, README, and map) was generated by an AI assistant
> (Claude) and lightly reviewed. Treat it as a sketch of pipeline shape, not
> as audited code.
>
> This is an **example**, not a recipe for a production application. It exists
> to show how a multi-stage agentic pipeline composes with `jigs`. The
> Cloudflare bindings (Workers AI, Vectorize, Cache API) are **mocked** in
> `src/bindings.rs` so the example runs locally without a Worker runtime.
>
> Do **not** copy this into a real Worker as-is. Things that are illustrative
> here but wrong for production:
>
> - The fake "embedding" function. Use `env.AI.run("@cf/baai/bge-base-en-v1.5", ...)`.
> - The static in-memory corpus in `vector_query`. Use Vectorize.
> - The `DefaultHasher`-based cache key. It is deterministic within a Rust
>   version but not across toolchains. Use a fixed-key SipHash or blake3.
> - The PII redactor only catches email-shaped tokens. Real redaction needs
>   a proper classifier.
> - The input filter is a tiny denylist. Real prompt-injection defenses are a
>   layered problem.
> - There is no rate limiting, no retries, no timeouts, no telemetry beyond
>   the `jigs` trace.

## What it shows

A request flows through:

1. `prepare` — `log_incoming`, `authenticate` (token → tenant).
2. `admit` (sync `Branch`) — `require_authenticated`, `input_filter`.
3. `lookup_cache` (async `Branch`) — Cache API hit short-circuits everything below.
4. `retrieve` — embed query, vector search, score-threshold + spam-tag filter, rerank.
5. `run` (the agent step) — `maybe_call_tool` falls back to `fetch` when context is thin, then `generate` calls the LLM and writes the result to the cache.
6. `finalize` — `pii_redact` on the way out.

The pipeline graph (`map.md` / `map.html`) is generated by `jigs::map` and
encodes which jigs are sync / async, and which can short-circuit.

## Run

```sh
cargo run -p jigs-example-cf-rag
```

Five paths are exercised: happy path, cache hit, unauth, blocked input, thin
context (tool fallback). Each prints the response plus the per-jig trace tree.

## Regenerate the map

```sh
JIGS_MAP=1 cargo run -p jigs-example-cf-rag
```

Writes `map.html` and `map.md` next to this README.
